# 录取数据标准对接规则

本规则专门用于把 `07_录取数据` 中后续爬到的原始数据，稳定回填到 `02_省份数据` 和 `03_院校库` 中。

# 一、数据分层

## 1. 原始层

位置：`07_录取数据`

特点：
- 原始抓取结果
- 允许是 CSV、Excel、JSON、HTML 摘录、PDF 转表
- 不在这个层里做人话总结
- 不直接给 AnythingLLM 使用

## 2. 省级汇总层

位置：`02_省份数据/省份/省份_录取数据总览.md`

承接内容：
- 控制线
- 一分一段 / 分数分布
- 本科普通批投档线
- 提前批 / 专项计划 / 公费项目入口
- 数据口径说明
- 官方来源链接

## 3. 学校层

位置：`03_院校库/省份/学校名.md`

承接内容：
- 与该校直接相关的最低分 / 位次 / 专业组
- 必须来自已清洗后的省级数据
- 不能绕过省级汇总层直接手填模糊结论

## 4. 导出层

位置：`11_发布导出`

承接内容：
- AnythingLLM 和网页应用使用的干净版知识文档
- 只能来自已验收的省级页、学校页、专业页

# 二、字段标准

## 省级最低必备字段

每个省的 `录取数据总览` 至少应逐步补齐：
- 年份
- 省份
- 考试模式
- 普通本科控制线
- 特殊类型控制线（如有）
- 一分一段 / 分数分布来源
- 本科普通批投档线来源
- 提前批 / 特招入口
- 官方来源链接
- 数据状态（已核/待补/待验）

## 学校录取数据字段

学校页里的录取表建议统一为：
- 年份
- 批次/类型
- 专业组 / 专业类 / 科类
- 选科要求
- 最低分
- 位次
- 来源
- 备注

## 状态标记

统一只用这 4 类：
- `已核`
- `待补`
- `待验`
- `约`

说明：
- 位次如果不是官方直接给出，而是由分数分布表换算，必须写 `约`
- 来源没有完全落仓时，一律 `待补`

# 三、回填顺序

每个省统一按下面顺序回填：

1. 先补控制线
2. 再补一分一段 / 分数分布
3. 再补本科普通批投档线
4. 再补提前批 / 特招 / 公费项目
5. 最后把高价值学校的表格回填进单校页

# 四、禁止事项

- 不允许把 `07_录取数据` 里的原始抓取直接喂给 AnythingLLM
- 不允许用 AI 猜测位次、专业组、最低分
- 不允许把不同年份、不同批次、不同口径的数据混在一张表里不标注
- 不允许把外省口径直接套到本省

# 四点五、2024 数据使用规则

如果当前仓库后续接入的录取数据主体是 `2024 年数据`，必须统一遵守下面这条说明：

- `2024 年录取数据仅供参考，不等于 2025/2026 年最终录取结果`
- 模型、页面文案、案例分析、学校页和专业页中，只要引用了 2024 年录取线、位次、专业组或投档结果，都应提醒用户：
  - `这是历史数据，只能用于参考趋势和大致定位`
  - `最终请以当年官方招生计划、控制线、投档线和志愿填报规则为准`

推荐统一提示语：

> 当前展示的录取数据以 2024 年历史数据为主，仅供参考，用于判断大致层级和趋势，不能直接等同于你今年的最终录取结果。

如果后续出现 2025、2026 年官方数据：

- 优先级必须高于 2024 年数据
- 页面中应明确区分“历史参考数据”和“当年正式数据”

# 五、回填映射规则

## 1. 回到省份页

- 控制线 -> `02_省份数据/省份/省份_录取数据总览.md`
- 一分一段 / 分数分布 -> 同页“官方数据入口”或“历年数据”部分
- 投档线 -> 同页“历年录取数据”部分

## 2. 回到学校页

只有满足下面条件才回填学校页：
- 该学校页已存在
- 数据已在省级页确认口径
- 能明确对应学校、专业组、选科要求

## 3. 回到导出页

只有满足下面条件才回填导出页：
- 已核
- 结构统一
- 来源可追溯
- 不包含抓取噪音和临时列

# 六、推荐工作流

- 第一步：爬虫把原始数据写入 `07_录取数据`
- 第二步：人工或脚本做轻清洗
- 第三步：把确认过的数据回填到 `02_省份数据`
- 第四步：从 `02_省份数据` 再回填到 `03_院校库`
- 第五步：从稳定页生成 `11_发布导出`

# 七、最重要的一句话

`07_录取数据` 是原始层，`02_省份数据` 才是知识层入口，`03_院校库` 是消费层，`11_发布导出` 是投喂层。

# 八、用户自行查询引导

当知识库只有历史数据，尤其是只有 `2024 年数据` 时，回答里应主动引导用户去查最新官方信息。

推荐优先引导到：

- 阳光高考
- 各省教育考试院 / 招生考试院官网
- 学信网

如果用户只是想快速搜索，也可以补充：

- 百度搜索

但要明确说明：

- `百度适合找入口，不适合直接把搜索结果当最终官方结论`
- `真正决定志愿填报的，仍然是官方文件和官方公布数据`
