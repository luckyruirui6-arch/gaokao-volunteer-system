# 结论

这两个仓库都能用，但用途不同：

- `gaokao-mentor-wisdom`
  最有价值的是 `分类结构`
- `zhangxuefeng-legency`
  最有价值的是 `高频问答题面`

它们都不适合原封不动直接放进你的产品知识库。

# 1. gaokao-mentor-wisdom 的可用点

从当前原始分类看，它把内容拆成：

- 专业选择
- 院校推荐
- 就业前景
- 人生哲理
- 学习建议
- 志愿填报策略

这个拆法对你的仓库有帮助，因为它很适合拿来做：

- 问答意图分类
- 首页入口分类
- AnythingLLM 工作区拆分参考
- 前端 FAQ 分类

# 2. gaokao-mentor-wisdom 的问题

- 更偏“语录库”而不是“事实库”
- 很多内容是观点表达，不带真实数据约束
- 如果直接导入 RAG，模型容易把金句当结论

# 建议改法

- 保留分类，不直接导入全部语录
- 把它改成 `问答分类标签体系`
- 只抽取那些能作为“表达框架”的内容

# 3. zhangxuefeng-legency 的可用点

这个仓库最有价值的是：

- 问题设计很贴近真实用户搜索
- 问答覆盖高考志愿、专业选择、考研、人生建议
- 很适合反推“用户到底会怎么问”

对你最有帮助的不是答案本身，而是 `问题集合`。

# 4. zhangxuefeng-legency 的问题

- 答案强风格化，很多句子像“强观点短答”
- 不区分“公开观点”和“已核验事实”
- 不适合直接拿来做产品底层回答

# 建议改法

- 保留 `q`
- 弱化 `a`
- 把答案改写成“问题模板 + 回答结构模板”
- 对高风险问题接入你仓库里的真实政策和院校数据

# 5. 这两个仓库在你项目里的最佳落点

- `gaokao-mentor-wisdom`
  放在 `05_张雪峰风格库/外部参考`

- `zhangxuefeng-legency`
  放在 `05_张雪峰风格库/外部参考`

原因是它们都更像“风格和问法参考”，不是“精确数据源”。

# 后续最值得吸收的内容

- 从 `gaokao-mentor-wisdom` 抽 `主题分类`
- 从 `zhangxuefeng-legency` 抽 `高频题面`
- 把这两部分改造成：
  - FAQ 标签体系
  - 系统提示词中的问答路由
  - 首页问答入口
